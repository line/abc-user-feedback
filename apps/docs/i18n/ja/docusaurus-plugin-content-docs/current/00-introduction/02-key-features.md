# 主な機能

ABC User Feedbackは、ユーザーフィードバックを効果的に収集、管理、分析するためのさまざまな機能を提供します。このドキュメントでは、コア機能を詳しく説明します。

## フィードバックタギングシステム

![Feedback Tag](/assets/01-feedback-tag.png)

フィードバックタギングシステムは、大量のユーザーフィードバックを体系的に分類・管理するためのコア機能です。

### 主な特徴

- **複数イシュー割り当て**: 各フィードバックに複数のイシューを割り当てて多次元的な分類が可能
- **カスタムイシュー作成**: プロジェクトの特性に合わせたカスタマイズされたイシューの作成・管理
- **イシューベースのフィルタリング**: イシュー別にフィードバックをフィルタリングして特定のトピックに集中
- **イシュー統計**: イシュー使用頻度とトレンド分析によるインサイトの導出
- **イシューカラーコーディング**: 視覚的な区別のためのイシュー別カラー指定

### 活用方法

1. 管理者パネルでイシューカテゴリとイシューを作成
2. 受信したフィードバックに関連するイシューを割り当て
3. イシュー別にフィードバックをフィルタリング・分析
4. イシュー使用パターンを通じて主要なイシューとトレンドを把握

## カンバンモード

![Issue Kanban](/assets/02-Issue-Kanban.png)

カンバンモードは、イシューグループを視覚的に管理し、ワークフローを最適化するための機能です。

### 主な特徴

- **直感的なドラッグアンドドロップ**: イシューステータス変更のためのシンプルなインターフェース
- **ステータス別列構成**: イシューの進行状況に応じた列の区別（例：To Do、進行中、完了）
- **ワークフローの視覚化**: チームの作業プロセスと進行状況を一目で把握
- **作業負荷管理**: 各ステータス別のイシュー数による作業負荷の監視
- **フィルタとソート**: さまざまな基準でカンバンボード内のイシューをフィルタリング・ソート

### 活用方法

1. カンバンモードビューを選択
2. ステータス別にイシューを確認・管理
3. ドラッグアンドドロップでイシューステータスを変更
4. チームのワークフローを最適化し、ボトルネックを特定

## イシュートラッカー連携

![Issue Tracker](/assets/03-issue-tracker.png)

イシュートラッカー連携は、フィードバックで発見された問題点や改善点を体系的に管理するための機能です。

### 主な特徴

- **ステータスインジケーター**: イシューの現在のステータスを視覚的に表示（新規、進行中、解決済みなど）
- **優先度設定**: イシューの重要度に応じた優先度の指定
- **外部システム連携**: 既存のイシュートラッカーシステム（JIRA、GitHub Issuesなど）と接続可能
- **担当者割り当て**: イシュー解決のための担当者指定
- **タイムライン追跡**: イシューステータス変更とコメント履歴の管理

### 活用方法

1. フィードバックからイシューを作成、または直接イシューを登録
2. イシューの詳細情報、優先度、担当者を設定
3. 外部イシュートラッカーと接続（オプション）
4. イシューの進行状況を監視・更新
5. 解決後、イシューを終了し、フィードバック提供者に通知（オプション）

## シングルサインオン（SSO）

![Single Sign-on](/assets/04-single-signon.png)

シングルサインオンは、企業環境での認証プロセスを簡素化し、セキュリティを強化する機能です。

### 主な特徴

- **OAuthサポート**: さまざまなOAuthプロバイダーによる認証サポート
- **企業ID統合**: 既存の企業IDシステムとのシームレスな統合
- **中央集約型ユーザー管理**: 単一の認証システムによるユーザーアクセス管理
- **セキュリティ強化**: 多要素認証と企業セキュリティポリシーの適用が可能
- **簡素化されたログイン体験**: ユーザーが追加のアカウントを作成する必要がない

### サポートするSSOプロバイダー

- Google
- Custom（標準OAuth 2.0およびOpenID Connectプロバイダー）

### 活用方法

1. 管理者設定でSSOプロバイダーを構成
2. 認証パラメータとリダイレクトURLを設定
3. ユーザー属性マッピングを構成
4. SSOログインを有効化してテスト

## ロールベースアクセス制御（RBAC）

![Role Management](/assets/05-role-management.png)

ロールベースアクセス制御は、ユーザーの権限を効果的に管理し、システムセキュリティを維持する機能です。

### 主な特徴

- **事前定義されたロール**: 管理者、アナリスト、ビューアーなどの基本ロールを提供
- **カスタムロール作成**: 組織構造に合わせたカスタマイズされたロールと権限設定
- **きめ細かい権限制御**: 機能別、データ別のアクセス権限設定
- **ロール割り当て管理**: ユーザー別のロール割り当てと変更
- **権限継承**: 階層的な権限構造のサポート

### デフォルトで提供されるロール

1. **システム管理者**: すべての機能と設定への完全なアクセス権限
2. **プロジェクト管理者**: 特定のプロジェクト内のすべての機能管理権限
3. **アナリスト**: フィードバックの閲覧、分析、イシュー指定の権限
4. **イシュー管理者**: イシューの作成、修正、ステータス変更の権限
5. **ビューアー**: フィードバックとダッシュボードの閲覧のみ可能な制限された権限

### 活用方法

1. 管理者パネルでロール管理メニューにアクセス
2. 必要に応じて新しいロールを作成、または既存のロールを修正
3. ユーザーに適切なロールを割り当て
4. ロール別の権限とアクセス範囲を定期的にレビュー

## ダッシュボード

![Dashboard](/assets/06-dashboard.png)

ダッシュボードは、フィードバックデータを視覚化して、重要なインサイトを一目で把握できるようにする機能です。

### 主な特徴

- **リアルタイム統計**: フィードバック数、イシューステータス、解決率などの主要指標をリアルタイムで表示
- **トレンド分析**: 時間に応じたフィードバックとイシューのトレンドグラフ
- **イシュー分布**: イシュー別のフィードバック分布の視覚化
- **感情分析**: フィードバックのポジティブ/ネガティブ/中立の感情分析（オプション機能）
- **カスタムダッシュボード**: ユーザー別のカスタマイズされたダッシュボード構成が可能

### 提供されるチャートとウィジェット

1. **フィードバックサマリーカード**: 総フィードバック数、新規フィードバック、処理済みフィードバックなどの主要指標
2. **時系列グラフ**: 日次/週次/月次のフィードバックトレンド
3. **イシュークラウド**: 頻繁に使用されるイシューの視覚化
4. **イシューステータスドーナツチャート**: イシューステータス別の分布
5. **最近のフィードバックリスト**: 最新のフィードバックを素早く確認
6. **パフォーマンス指標**: 応答時間、解決時間などのパフォーマンス測定指標

### 活用方法

1. ダッシュボードページにアクセス
2. 期間とフィルタ設定を通じてデータ範囲を調整
3. 主要指標とトレンドを分析
4. インサイトベースの意思決定とアクションアイテムの導出
5. 必要に応じて詳細レポートをエクスポート

## 追加機能

ABC User Feedbackは、上記で説明した主要機能に加えて、以下の追加機能も提供します：

### API統合

- RESTful APIによる外部システムとの連携
- プログラムによるフィードバック収集と管理

### Webhook

- 主要イベント発生時に外部システムに通知
- 自動化されたワークフローの構築をサポート

### 画像保存統合

- S3互換ストレージによるユーザー提出画像の管理
- フィードバックにスクリーンショットと画像を添付する機能

### データエクスポート

- CSV、Excel形式でのフィードバックデータのエクスポート
- カスタマイズされたレポートの生成

### 多言語サポート

- さまざまな言語でのインターフェース提供
- 国際的なチームのための多言語フィードバック管理
