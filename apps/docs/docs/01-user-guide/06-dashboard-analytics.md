# 대시보드

**대시보드**는 피드백과 이슈 데이터를 실시간으로 시각화하여 중요한 인사이트를 한눈에 파악할 수 있게 해주는 핵심 기능입니다. 대시보드의 모든 차트와 위젯을 활용하여 데이터 기반 의사결정을 내리는 방법을 설명합니다.

---

## 대시보드 개요

### 대시보드의 목적

대시보드는 다음 목적으로 사용됩니다:

- **실시간 모니터링**: 피드백 수집 현황과 이슈 처리 상태 실시간 추적
- **트렌드 분석**: 시간에 따른 피드백 및 이슈 변화 패턴 파악
- **문제점 식별**: 급증하는 이슈나 문제 영역 조기 발견
- **성과 측정**: 이슈 해결률, 응답 시간 등 KPI 모니터링
- **데이터 기반 의사결정**: 객관적 데이터를 바탕으로 한 전략 수립

### 접근 방법

1. 상단 메뉴에서 **Dashboard** 탭을 클릭합니다

---

## 통계 카드

대시보드 상단에는 핵심 지표가 카드 형태로 요약되어 표시됩니다.

### 카드별 설명

| 카드명                      | 설명                  | 표시 정보               |
| --------------------------- | --------------------- | ----------------------- |
| **Total Feedback**          | 전체 누적 피드백 수   | 총 개수 + 증감률        |
| **Today's Feedback**        | 당일 등록된 피드백 수 | 당일 개수 (증감률 없음) |
| **Yesterday's Feedback**    | 전일 등록된 피드백 수 | 전일 개수 (증감률 없음) |
| **Last 7 Days' Feedback**   | 지난 7일간 피드백 수  | 7일 총합 + 증감률       |
| **Last 30 Days' Feedback**  | 지난 30일간 피드백 수 | 30일 총합 + 증감률      |
| **Total Issues**            | 전체 누적 이슈 수     | 총 개수 (증감률 없음)   |
| **Issue Registration Rate** | 신규 이슈 등록률      | 비율 (증감률 없음)      |
| **Today's Issues**          | 당일 생성된 이슈 수   | 당일 개수 (증감률 없음) |

### 날짜 필터 설정

대시보드 우측 상단의 **Date** 필터를 통해 분석 기간을 설정할 수 있습니다.

### 뷰 설정

우측 상단의 **View** 드롭다운에서 표시할 통계 카드를 조정할 수 있습니다.

### 증감률 표시

일부 카드에는 이전 기간 대비 증감률이 표시됩니다:

- **녹색 ↑**: 증가 (예: +47.3%)
- **빨간색 ↓**: 감소 (예: -15.2%)
- **회색**: 변화 없음 또는 해당 없음

## Feedback Trends 차트

피드백 수집 추이를 시간에 따라 시각화한 영역 차트입니다.

### 차트 특징

- **시간축**: X축에 날짜별 구간 표시
- **수량축**: Y축에 피드백 수 표시
- **영역 차트**: 누적된 피드백 양을 면적으로 표현
- **피크 식별**: 급증한 시점을 시각적으로 쉽게 파악

### 필터 기능

차트 우측 상단의 **Filter** 버튼을 통해 다음을 설정할 수 있습니다:

- **채널별 필터링**: 특정 채널의 피드백만 표시
- **다중 채널 선택**: 여러 채널을 동시에 비교 분석
- **전체/부분 보기**: 필요한 데이터만 선택적으로 표시

### 활용 방법

1. **급증 시점 분석**: 피드백이 급격히 증가한 날짜 확인
2. **계절성 파악**: 주기적인 증감 패턴 식별
3. **채널별 비교**: 어떤 채널에서 더 많은 피드백이 수집되는지 파악
4. **이벤트 영향 측정**: 특정 이벤트나 배포 이후의 피드백 변화 추적

## Issue Trends 차트

이슈 생성 추이를 시간에 따라 시각화한 영역 차트입니다.

### 차트 분석

- **이슈 발생 패턴**: 어떤 시기에 이슈가 집중적으로 생성되는지 파악
- **피드백과의 상관관계**: Feedback Trends와 비교하여 피드백 대비 이슈 전환률 분석
- **문제 발생 주기**: 반복적으로 발생하는 이슈 패턴 식별

### 활용 예시

```
피드백 급증 → 이슈 증가 (정상적인 패턴)
피드백 안정 → 이슈 급증 (잠재 문제 발생)
피드백 증가 → 이슈 감소 (개선 효과 확인)
```

## Issue Status Overview

현재 이슈들의 상태별 분포를 시각화한 차트입니다.

### 차트 유형

#### Bar 차트

- **막대 그래프**: 각 상태별 이슈 수를 막대 높이로 표현
- **비교 용이**: 상태 간 수량 차이를 직관적으로 파악

#### Pie 차트

우측 상단의 **Pie** 버튼을 클릭하여 전환할 수 있습니다.

- **원형 차트**: 전체 이슈 중 각 상태의 비율을 시각화
- **비율 파악**: 전체 대비 각 상태의 점유율 확인

### 상태별 분석

| 상태            | 분석 포인트                    |
| --------------- | ------------------------------ |
| **New**         | 신규 이슈 적체 여부            |
| **On Review**   | 검토 단계의 처리 속도          |
| **In Progress** | 실제 작업 중인 이슈 부하       |
| **Resolved**    | 해결 완료된 이슈 비율          |
| **On Hold**     | 보류된 이슈의 원인과 해결 방안 |

## Top Issues 테이블

가장 많은 피드백이 연결된 상위 이슈들을 테이블 형태로 표시합니다.

### 테이블 구성

| 컬럼               | 설명                  | 정렬 가능 |
| ------------------ | --------------------- | --------- |
| **No**             | 순위                  | -         |
| **Issue**          | 이슈 제목 (링크)      | -         |
| **Status**         | 현재 상태 (배지 표시) | -         |
| **Feedback Count** | 연결된 피드백 수      | ✅        |

### 설정 옵션

#### Top N 선택

우측 상단의 **Top 5** 드롭다운에서 표시할 이슈 수를 변경할 수 있습니다:

- Top 5, Top 10, Top 20 등

#### 상태 필터

**Filter** 버튼을 통해 특정 상태의 이슈만 표시할 수 있습니다:

### 활용 방법

1. **우선순위 설정**: 피드백 수가 많은 이슈를 우선 처리
2. **영향도 평가**: 사용자에게 미치는 영향 범위 파악
3. **리소스 할당**: 중요도가 높은 이슈에 인력 집중
4. **상태 모니터링**: 상위 이슈들의 해결 진행 상황 추적

## Issue Comparison 차트

여러 이슈의 피드백 추이를 동시에 비교 분석할 수 있는 차트입니다.

### 차트 특징

- **다중 라인 차트**: 각 이슈별로 다른 색상의 라인으로 표시
- **범례 표시**: 우측에 이슈별 색상 구분 범례
- **시간 동기화**: 모든 이슈의 추이를 동일한 시간축에서 비교

### 이슈 선택

우측의 **Selected Issue** 패널에서 비교할 이슈들을 선택할 수 있습니다:

- **체크박스**: 각 이슈별로 표시/숨김 토글
- **색상 구분**: 각 이슈마다 고유한 색상 할당
- **실시간 업데이트**: 선택 변경 시 차트 즉시 반영

### 분석 활용

#### 이슈 간 상관관계 분석

```
이슈 A 증가 → 이슈 B 감소 (대체 관계)
이슈 A 증가 → 이슈 B 증가 (연쇄 관계)
```

#### 해결 효과 측정

```
해결 조치 전후의 이슈별 피드백 변화 비교
특정 업데이트가 여러 이슈에 미친 영향 분석
```

#### 우선순위 재평가

```
과거에는 높았지만 현재 감소한 이슈
최근 급증하고 있는 새로운 이슈
```

## 문제 해결

### 자주 발생하는 문제

| 문제 유형                  | 원인 및 해결 방법                      |
| -------------------------- | -------------------------------------- |
| 차트가 로드되지 않음       | 네트워크 확인, 브라우저 새로고침       |
| 데이터가 업데이트되지 않음 | 캐시 클리어, 페이지 새로고침           |
| 필터가 적용되지 않음       | 필터 설정 재확인, 브라우저 호환성 점검 |
| 차트가 깨져서 보임         | 브라우저 확대/축소 비율 100%로 조정    |
| 날짜 필터 오류             | 선택한 날짜 범위 유효성 확인           |

### 지원 요청

문제가 지속되는 경우:

1. **브라우저 정보 확인**: 브라우저 종류 및 버전
2. **오류 메시지 캡처**: 개발자 도구의 콘솔 오류
3. **재현 단계 기록**: 문제가 발생하는 정확한 조건
4. [GitHub Issues](https://github.com/line/abc-user-feedback/issues)에 상세 정보와 함께 문의
