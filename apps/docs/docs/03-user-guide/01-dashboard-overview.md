# 대시보드 사용법

ABC User Feedback의 대시보드는 피드백과 이슈의 전체적인 현황을 한눈에 볼 수 있는 홈 화면입니다. 실시간 데이터와 다양한 차트를 통해 프로젝트의 상태를 효과적으로 모니터링할 수 있습니다.

## 화면 구성

### 상단 네비게이션

#### 프로젝트 선택

- **위치**: 좌측 상단 드롭다운
- **기능**: 현재 작업 중인 프로젝트/환경 선택
- **사용법**: 드롭다운을 클릭하여 분석하고자 하는 프로젝트를 선택합니다

#### 메뉴 탭

대시보드 상단에 위치한 주요 메뉴들:

- **Dashboard**: 현재 페이지 (대시보드)
- **Feedback**: 피드백 관리 페이지로 이동
- **Issue**: 이슈 관리 페이지로 이동
- **Settings**: 설정 페이지로 이동

#### 우측 컨트롤

- **📅 Date**: 분석 기간 설정 버튼
- **👁️ View**: 뷰 옵션 선택 버튼

### 기간 설정하기

1. **Date 버튼 클릭**
2. 시작 날짜와 종료 날짜를 선택
3. 설정된 기간에 따라 모든 통계와 차트가 자동으로 업데이트됩니다

## 통계 카드 활용법

대시보드 상단의 통계 카드들은 피드백과 이슈의 핵심 지표를 실시간으로 보여줍니다. 각 카드에는 전 기간 대비 증감률이 표시되며, 카드에 마우스를 올리면 해당 지표에 대한 상세한 설명을 툴팁으로 확인할 수 있습니다.

### 증감률 해석하기

- **녹색 (▲)**: 이전 기간 대비 증가
- **빨간색 (▼)**: 이전 기간 대비 감소
- **백분율**: 변화량의 크기를 나타냄

:::tip 통계 카드 활용 팁
증감률 변화의 의미를 컨텍스트에 맞게 해석하세요. 피드백 증가는 관심도 상승을, 이슈 증가는 문제 발생 증가를 의미할 수 있습니다.
:::

## 차트 및 분석

### Feedback Trends 차트 사용법

피드백 접수량의 시간별 변화를 라인 차트로 보여줍니다.

**차트 읽기:**

- **X축**: 시간 (날짜별)
- **Y축**: 피드백 수
- **다중 라인**: 각 채널별로 다른 색상의 라인

**필터링 기능:**

1. **Filter 버튼** 클릭
2. 특정 채널만 선택하여 표시
3. 채널별 성과 비교 분석 가능

**활용 방법:**

- 피크 시간대 파악으로 고객 활동 패턴 분석
- 채널별 성과 비교
- 특정 이벤트나 업데이트 후 변화 확인

### Issue Trends 차트 사용법

이슈 생성량의 시간별 변화를 라인 차트로 보여줍니다.

**특징:**

- Feedback Trends와 동일한 시간축 사용
- 피드백 대비 이슈 발생 패턴 비교 가능

**활용 방법:**

- 이슈 발생 패턴 분석
- 피드백 트렌드와 비교하여 상관관계 파악
- 문제 발생 시점 추적

### Issue Status Overview 차트 사용법

이슈의 상태별 분포를 시각화합니다.

**차트 유형 전환:**

- **Bar**: 세로 막대 차트 (기본)
- **Pie**: 원형 차트로 전환 가능

**상태별 의미:**

- **New**: 새로 생성된 이슈
- **On Progress**: 진행 중인 이슈
- **In Progress**: 작업 중인 이슈
- **Resolved**: 해결된 이슈
- **On Hold**: 보류된 이슈

**분석 포인트:**

- New 상태 비율이 높으면 → 처리 인력 보강 검토
- Resolved 비율이 낮으면 → 해결 프로세스 점검
- On Hold 이슈가 많으면 → 보류 사유 분석

### Top Issues 테이블 사용법

피드백 수가 많은 상위 이슈들을 테이블 형태로 표시합니다.

**테이블 구성:**

- **No**: 순위
- **Issue**: 이슈 제목
- **Status**: 현재 상태
- **Feedback Count**: 연결된 피드백 수

**정렬 및 필터:**

1. **Top N 드롭다운**: 표시할 상위 이슈 개수 조정 (기본값: Top 5)
2. **Filter 버튼**: 상태별, 기간별 필터링

**우선순위 결정:**

1. Feedback Count가 높은 이슈부터 우선 처리
2. 상태와 피드백 수를 종합적으로 고려
3. On Hold 상태 이슈들의 재검토

### Issue Comparison 차트 사용법

여러 이슈의 트렌드를 시간별로 비교합니다.

**기능:**

- 다양한 색상으로 각 이슈 구분
- 시간대별 이슈별 피드백 변화 추이
- 특정 이슈만 선택하여 비교 가능

**Filter 사용법:**

1. **Filter 버튼** 클릭
2. 비교하고 싶은 이슈들만 선택
3. 불필요한 이슈는 체크 해제

**활용 방법:**

- 이슈간 상대적 중요도 비교
- 시간에 따른 이슈 변화 패턴 분석
- 리소스 배분 결정을 위한 데이터 활용

## 대시보드 활용 가이드

### 효과적인 모니터링 워크플로우

#### 1단계: 전체 현황 파악

1. 통계 카드에서 급격한 증감률 확인
2. 이상 수치가 있으면 해당 기간 상세 분석
3. Total 수치와 기간별 수치 비교

#### 2단계: 트렌드 분석

1. **Feedback Trends**에서 피크 시간대 확인
2. **Issue Trends**와 비교하여 상관관계 파악
3. 특정 패턴이나 이상 현상 식별

#### 3단계: 우선순위 결정

1. **Top Issues**에서 처리 우선순위 결정
2. **Issue Status Overview**에서 전체 처리 현황 점검
3. 긴급도와 중요도 매트릭스 활용

#### 4단계: 액션 플랜 수립

1. **Issue Comparison**으로 리소스 배분 계획
2. 다음 모니터링 주기 설정
3. 알림 및 보고 체계 확인

### 기간별 분석 전략

#### 일일 모니터링

- Today's/Yesterday's 카드 중심으로 확인
- 급격한 변화 시 즉시 대응
- 실시간 이슈 발생 모니터링

#### 주간 분석

- Last 7 Days 카드 + Feedback Trends 활용
- 주간 패턴 파악 및 예측
- 주말/평일 차이 분석

#### 월간 리포트

- Last 30 Days 카드 + 모든 차트 종합 분석
- 월별 성과 평가
- 장기 트렌드 식별

### 데이터 해석 시 주의사항

#### 컨텍스트 고려

- 마케팅 캠페인, 제품 업데이트, 이벤트 등의 영향
- 계절성이나 주기적 패턴 고려
- 외부 요인 (경쟁사, 시장 상황 등) 반영

#### 상대적 해석

- 절대 수치보다는 변화율과 패턴에 집중
- 다른 지표들과의 상관관계 분석
- 과거 동일 기간과의 비교

:::warning 주의사항

- 급격한 증감률 변화 시 원인 분석 필요
- 기간 설정에 따라 데이터 해석이 달라질 수 있음
- 실시간 데이터이므로 페이지 새로고침으로 최신 정보 확인
  :::

:::info 추가 정보
대시보드의 모든 차트와 데이터는 설정된 날짜 범위와 선택된 프로젝트를 기반으로 표시됩니다. 정확한 분석을 위해 적절한 기간과 프로젝트를 선택했는지 확인하세요.
:::
